+ http의 요청 방식
 - Http request 메시지를 서버로 보낼 때 요청 데이터를
 전달하는 방식을 나타낸다.

 - 요청 메시지에는 반드시 요청 메소드가 정의되어 있어야 한다.
(지정하지 않으면 기본 get 방식을 사용한다.)

- get 방식, post 방식을 주로 사용한다.



+ get 방식
 - 요청데이터를 쿼리스트링 형태로 url의 마지막에 ? 를 붙이고 그 뒤에 포함한다.

	ex)http://localhost:8088/search?keyword=java
	
** 쿼리스트링, query String http 통신(웹 통신)에서 전달데이터의 변수명과 값을 표현하는 방식
 '변수명 = 값' 쌍의 형태로 표현한다
 여러 개의 변수를 나타낼 때에는 각 데이터쌍 사이에 &를 적어서 구분한다.

	ex) id=abc&pass=123


** get 방식은 주로 서버의 데이터를 조회하려고 할 때 사용한다.
	->select
	데이터가 없거나, 짧을 때 사용한다.


+ post 방식
 - 전달데이터를 쿼리스트링 형태로 요청메시지의 body 영영에 포함시켜 전달한다.

 - get 방식과는 다르게 url에 전달하려는 데이터가 보이지 않는다.

** post 방식은 주로 서버의 데이터를 삽입/갱신/삭제하려고 할때
 -> insert, update, delete
	데이터가 길 때 사용한다.
	민감데이터일 경우 post방식을 사용한다.(비밀번호)
