+ Http 요청의 Get, Post 메소드

 - GET 메소드는 서버의 자원을 조회할 때 사용한다.(SELECT)
 - POST 메소드는 서버의 자원을 삽입, 수정, 삭제할 때 사용한다.
	(INSERT UPDATE DELETE)
	
 - 전달파라미터가 짧은(단순한) 데이터이거나 없으면 GET을 사용한다
 - 전달파라미터 긴(복잡한) 데이터이거나 민감한 정보라면 POST를 사용한다

 - GET 메소드는 URL 뒤에 쿼리스트링으로 전달파라미터를 붙여서 서버로 전달한다.
 - URL에 전달하려는 데이터가 노출된다

 - POST는 요청메시지의 BODY 영영에 쿼리스트링으로 전달파라미터를 추가해 서버로 전달

 - 전달하려는 데이터가 겉으로 노출되지 않는다


-----------------------------------------------------------------


 + Http 요청이 Get 메소드로 전달되는 상황
 - 단순히 url만 요청했을 때(브라우저 주소입력창에 url입력하고 엔터)
 - <form>태그에 method를 get으로 설정하고 submit했을 때

 - <a> 링크를 클릭했을 때

 - location.href = "URL" 로 페이지 이동했을 때

 - <link> 나 <script> 태그로 자원(파일)을 로드했을 때


 + Http 요청이 Post 메소드로 전달되는 상황
  - <form> 태그에 method를 post으로 설정하고 submit했을 때
