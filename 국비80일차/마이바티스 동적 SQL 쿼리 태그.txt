+ 마이바티스 동적 SQL 쿼리, Dynamic SQL
 - 동적 변환되는 SQL 쿼리를 작성할 수 있도록 도와주는 마이바티스 태그

 - <select> <insert> <update> <delete> 태그 안에서 사용한다.

 - <if>, <choose>, <trim>, <foreach> 태그를 제공한다


+ <if> 태그
 - test 속성을 이용하여 조건문을 지정하여 사용하는 조건문 태그

 - 조건문의 결과에 따라 쿼리의 추가 여부가 결정된다

 - test속성의 값은 "true" 또는 "false"로 계산되게끔 작성해야 한다

+ <choose> 태그

 - 자식태그로 <when>, <otherwise> 태그를 같이 사용한다

 - if ~ else if ~ else 구문과 비슷한 제어문 태그

 - <choose> 태그에는 속성을 작성하지 않는다

 - <when> 태그에 test 속성으로 조건문을 작성한다

 - <when> 태그들의 마지막으로 <otherwise>태그를 사용할 수 있다
 - <otherwise>태그에는 test 속성을 작성하지 않는다

 ex) <choose>
	<when test= "조건문1">
		추가될 SQL1
	</when>

	<when test= "조건문2">
		추가될 SQL2
	</when>

	. . . . .


	<otherwise>
		추가될 SQL3
	</otherwise>
 </choose>	


+ <foreach> 태그
 - sql 구문이 반복적으로 추가되고 동적으로 바뀌는 구문을 생성할 때 사용하는 태그

 - 배열/해시맵 타입의 파라미터를 처리할 때 유용하다

 - 마이바티스의 파라미터값은 hashmap으로 처리하는 것이 좋다
 - 배열이나 리스트를 해시맵에 저장(put)하여 사용한다

 - 속성
	collection: 반복을 수행할 객체(리스트, 배열)을 저장한다
	item: 반복될 때마다 사용하는 요소의 이름을 지정한다

	open: 구문의 첫부분에 사용할 단어를 지정한다
	close: 구문의 끝부분에 사용할 단어를 지정한다

	separator: 반복되는 구문 사이사이에 추가할 단어를 지정한다

 ** checkbox 선택항목에 대한 조건처리에 유용하게 사용된다

+ <trim> 태그

 - where 절이나 update의 set절에서 사용하는 태그

 - 전달파라미터가 전달되지 않아 동적쿼리가 추가되어지 않아야 하는 상황에서
	where절의 and 나 or 키워드의 충돌
	set 절의 마지막 ',' 를 제거할 때 사용한다

 - 속성
	prefix: 접두어, 맨 앞쪽에 반드시 추가될 문자열
	prefixOverries: 접두어로 설정한 값과 겹쳐지면 지울 문자열

	suffix: 접미어, 맨 뒤쪽에 반드시 추가될 문자열
	suffixOverrides: 접미어로 설정한 값과 겹쳐지면 지울 문자열

 ** prefix나 suffix를 설정하지 않으면 각각 Overrides는 첫 값, 
 마지막 값을 판단하여 제거한다
